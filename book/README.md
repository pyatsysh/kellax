# kellax by solved problems

A short book that teaches numerical continuation by *tracing real bifurcation
diagrams* with [kellax](../README.md). Every chapter is one worked problem: the
theory emerges from the problem, the runnable script in [`examples/`](../examples)
is the source of truth, and the figure is regenerated by running it.

## Principles

- **Theory from problems.** No chapter opens with abstract machinery; it opens
  with an equation whose solution branch we want, and the method appears where the
  naive approach breaks.
- **Every number is validated.** Each chapter states the known answer (an analytic
  fold location, a literature value) and the script checks kellax against it. The
  printed numbers in each chapter are real output.
- **The code is the source of truth.** Chapters show short excerpts; the full,
  runnable version lives in `examples/<name>.py` and writes `figures/<name>.png`.
  Regenerate everything with the commands under *Running* below.
- **Matrix-free where it matters.** Small ODE/normal-form problems use the dense
  engine; discretised PDE fields use the matrix-free GMRES-on-JVP engine.

## What each problem teaches

- **The fold** — why naive parameter-stepping fails, and how the bordered
  pseudo-arclength system walks through a turning point; then Moore–Spence
  refinement pins the fold to Newton precision.
- **The cusp** — a fold is codimension-1, so in two parameters it is a *curve*;
  `track_fold` traces the whole fold set — both arms — in a single continuation
  that passes smoothly through the cusp.
- **Bratu–Gelfand** — the same machinery on a discretised boundary-value problem;
  autodiff means the discretisation is the only modelling you do, and stability
  (cool vs hot branch) is read off the Jacobian's spectrum.
- **Scaling up** — the same trace run matrix-free (GMRES on Jacobian–vector
  products), so it keeps working when the Jacobian no longer fits in memory.
- **Snaking** — that engine on a pattern-forming PDE, one connected branch
  wiggling through dozens of folds as localized patterns grow.

## Running

```bash
uv venv .venv && uv pip install --python .venv/bin/python -e .
uv pip install --python .venv/bin/python matplotlib
python examples/cubic_fold.py         # -> figures/cubic_fold.png  (+ validation to stdout)
python examples/cusp.py               # -> figures/cusp.png
python examples/bratu.py              # -> figures/bratu.png
python examples/bratu_matrixfree.py   # -> figures/bratu_matrixfree.png
python examples/swift_hohenberg.py    # -> figures/swift_hohenberg.png
python examples/cstr.py               # -> figures/cstr.png
python examples/predator_prey.py      # -> figures/predator_prey.png
python examples/bratu_2d.py           # -> figures/bratu_2d.png
```
